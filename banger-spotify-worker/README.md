# Banger Spotify Worker

A Python Cloudflare Worker using FastAPI that handles Spotify OAuth token storage and refresh on behalf of the Go Discord bot.

## How to Run

First ensure that `uv` is installed:
https://docs.astral.sh/uv/getting-started/installation/#standalone-installer

Run locally (picks up secrets from `.dev.vars`):

```bash
uv run pywrangler dev
# or
npm run dev
# or
npx wrangler dev
```

Deploy to Cloudflare:

```bash
uv run pywrangler deploy
# or
npm run deploy
# or
npx wrangler deploy
```

To test on a specific port:

```bash
uv run pywrangler dev --port 8989
```

Then verify endpoints via curl or Postman against `http://127.0.0.1:8989/`, e.g.:

```bash
curl "http://127.0.0.1:8989/auth-url?user_id=271828182845"
```

## Local Secrets

Local environment variables are set in a `.dev.vars` file in this directory (automatically picked up by Wrangler; never commit this file):

```
SPOTIFY_CLIENT_ID=...
SPOTIFY_CLIENT_SECRET=...
REDIRECT_URI=http://localhost:8787/callback
```

| Variable | Description |
| --- | --- |
| `SPOTIFY_CLIENT_ID` | Client ID for Spotify API |
| `SPOTIFY_CLIENT_SECRET` | Client secret for Spotify API |
| `REDIRECT_URI` | Callback URL registered in the Spotify Developer Dashboard |

In production, these are set as Wrangler secrets (`npx wrangler secret put <NAME>`).

## KV Storage

The worker uses a KV namespace bound as `SPOTIFY_TOKENS` (configured in `wrangler.jsonc`).

| Key | Value | Set by |
| --- | --- | --- |
| `__signing_key__` | Random 32-byte hex string | Worker on first `/auth-url` call (self-generated, never manually set) |
| `{discord_user_id}` | Spotify token JSON with `expires_at` field | Worker on `/callback` and `/token` auto-refresh |

## Endpoints

| Method | Endpoint | Auth | Description |
| --- | --- | --- | --- |
| `GET` | `/auth-url?user_id={id}` | CF Access service token | Returns a signed Spotify OAuth URL for the given Discord user ID |
| `GET` | `/callback?code=&state=` | Public (CF bypass) + HMAC state | Exchanges authorization code for tokens; stores in KV |
| `GET` | `/token/{user_id}` | CF Access service token | Returns stored token; auto-refreshes if within 60s of expiry |
| `POST` | `/refresh/{user_id}` | CF Access service token | Force-refreshes the token regardless of expiry |

`user_id` is the Discord user ID (string). It is used as the KV storage key for the user's Spotify token.

## Security

In production, every endpoint except `/callback` is protected by Cloudflare Zero Trust Access. Requests must include a valid CF Access service token:

```
CF-Access-Client-Id: <client-id>
CF-Access-Client-Secret: <client-secret>
```

The `/callback` endpoint is excluded from CF Access (bypass policy) so Spotify's OAuth redirect can reach it without credentials. It is instead protected by an HMAC-signed `state` parameter (CSRF protection). The signing key is self-generated by the worker on first use and stored in KV under `__signing_key__`.
